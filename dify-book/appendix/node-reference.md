# 节点速查表

## 基础节点

### 开始（Start）
- **作用**：定义 Workflow 的输入变量
- **配置**：变量名、类型、是否必填
- **使用频率**：★★★★★（每个 Workflow 都有）

### 结束（End）
- **作用**：定义 Workflow 的输出
- **配置**：输出变量名、来源
- **使用频率**：★★★★★

### LLM
- **作用**：调用大语言模型
- **配置**：模型、System/User 提示词、温度
- **使用频率**：★★★★★（核心节点）

### 知识检索（Knowledge Retrieval）
- **作用**：从知识库检索内容
- **配置**：查询变量、知识库、检索数量
- **使用频率**：★★★★☆

## 逻辑节点

### 条件分支（IF/ELSE）
- **作用**：根据条件走不同路径
- **配置**：条件表达式、多分支
- **使用频率**：★★★★☆

### 问题分类器
- **作用**：用 AI 判断输入类别
- **配置**：分类定义、模型
- **使用频率**：★★★☆☆

### 迭代（Iteration）
- **作用**：遍历数组执行操作
- **配置**：输入数组、子流程
- **使用频率**：★★★☆☆

## 处理节点

### 参数提取器
- **作用**：从文本提取结构化信息
- **配置**：参数定义、模型
- **使用频率**：★★★☆☆

### 模板转换
- **作用**：用 Jinja2 格式化输出
- **配置**：模板内容
- **使用频率**：★★★☆☆

### 代码执行
- **作用**：执行 Python/JS 代码
- **配置**：代码、输入输出变量
- **注意**：代码必须包裹在 `main()` 函数中，通过参数接收输入、通过 `return` 返回输出。不支持裸代码直接运行
- **使用频率**：★★☆☆☆

## 集成节点

### HTTP 请求
- **作用**：调用外部 API
- **配置**：URL、方法、Headers
- **使用频率**：★★☆☆☆

### Agent
- **作用**：让 AI 自主调用工具
- **配置**：模型、策略、可用工具
- **使用频率**：★★★☆☆

### 变量赋值器（Variable Assigner）
- **作用**：将数据写入指定变量（Workflow 中用于合并分支、Chatflow 中用于写会话变量）
- **配置**：目标变量、数据来源、操作模式（覆盖/追加/清空）
- **使用频率**：★★★☆☆

## Chatflow 专属节点

### 直接回复（Answer）
- **作用**：在 Chatflow 中向用户发送回复
- **配置**：回复内容（支持变量引用）
- **说明**：相当于 Workflow 的「结束」节点，但不会终止对话，会等待用户下一轮输入
- **使用频率**：★★★★★（Chatflow 必备）

### 变量赋值器在 Chatflow 中的用法
- **说明**：同上面的「变量赋值器」节点。在 Chatflow 中主要用于写入会话变量，实现跨轮次记忆——是 Chatflow"记住上下文"的关键
- **使用频率**：★★★★☆（Chatflow 常用）

### 会话变量（Conversation Variable）
- **作用**：在多轮对话中持久存储数据（非节点，是 Chatflow 的全局设置）
- **配置**：变量名、类型（String/Number/Array/Object 等）
- **说明**：在 Chatflow 画布顶部的「会话变量」按钮中管理。普通变量每轮重置，会话变量跨轮次保留
- **使用频率**：★★★★☆（Chatflow 常用）

# 节点速查表

## 基础节点

### 开始（Start）
- **作用**：定义 Workflow 的输入变量
- **配置**：变量名、类型、是否必填
- **使用频率**：★★★★★（每个 Workflow 都有）

### 结束（End）
- **作用**：定义 Workflow 的输出
- **配置**：输出变量名、来源
- **使用频率**：★★★★★

### LLM
- **作用**：调用大语言模型
- **配置**：模型、System/User 提示词、温度
- **使用频率**：★★★★★（核心节点）

### 知识检索（Knowledge Retrieval）
- **作用**：从知识库检索内容
- **配置**：查询变量、知识库、检索数量
- **使用频率**：★★★★☆

## 逻辑节点

### 条件分支（IF/ELSE）
- **作用**：根据条件走不同路径
- **配置**：条件表达式、多分支
- **使用频率**：★★★★☆

### 问题分类器
- **作用**：用 AI 判断输入类别
- **配置**：分类定义、模型
- **使用频率**：★★★☆☆

### 迭代（Iteration）
- **作用**：遍历数组执行操作
- **配置**：输入数组、子流程
- **使用频率**：★★★☆☆

## 处理节点

### 参数提取器
- **作用**：从文本提取结构化信息
- **配置**：参数定义、模型
- **使用频率**：★★★☆☆

### 模板转换
- **作用**：用 Jinja2 格式化输出
- **配置**：模板内容
- **使用频率**：★★★☆☆

### 代码执行
- **作用**：执行 Python/JS 代码
- **配置**：代码、输入输出
- **使用频率**：★★☆☆☆

## 集成节点

### HTTP 请求
- **作用**：调用外部 API
- **配置**：URL、方法、Headers
- **使用频率**：★★☆☆☆

### Agent
- **作用**：让 AI 自主调用工具
- **配置**：模型、策略、可用工具
- **使用频率**：★★★☆☆

### 变量聚合器
- **作用**：合并多分支输出
- **配置**：输入变量列表
- **使用频率**：★★☆☆☆

# 第 1 章：从聊天机器人到 AI 工作流

> 在开始动手之前，让我们花几分钟理解一下 Dify 到底是什么，以及它和你可能已经用过的 ChatGPT 有什么不同。

## ChatGPT 改变了什么

2022 年底，ChatGPT 横空出世，让全世界见识到了大语言模型的威力。短短两年，AI 的能力突飞猛进——不仅能对话，还能写作、调用工具、甚至操作电脑。

但对于企业来说，**"能做"和"做得好"是两回事**。

让 AI 回答一个问题很容易。让 AI 稳定地、按照你的业务规则、基于你的企业数据、持续地处理成千上万个问题——这需要的不是更好的提示词，而是一整套系统。

知识库要有人管理，工作流要有人设计，业务系统要能接入，运行效果要能监控，出了问题要能迭代优化。

**这些事情，光靠一个聊天框是搞不定的。**

## 从「对话」到「应用」

真正的业务价值，不在于和 AI 聊天，而在于让 AI 按照你设计的流程，稳定地完成任务。

这就是从"AI 对话"到"AI 应用"的跨越。

一个 AI 应用有什么特点？

1. **有明确的输入和输出**：不是漫无目的的聊天，而是"给它什么，得到什么"
2. **有固定的处理流程**：不是每次都随机回答，而是按照你设计的步骤执行
3. **可以接入外部数据**：不只依赖模型的训练知识，还能查询你的文档、数据库、API
4. **可以被其他系统调用**：不只是人在界面上操作，还能通过 API 嵌入到现有业务系统

**Dify 就是帮你构建这种 AI 应用的平台。**

## Dify 的定位

Dify 的全称是"Define + Modify"——定义，然后持续改进。它的设计理念是：

**让每个人都能构建生产级的 AI 应用，无论是否有编程背景。**

Dify 提供了什么？

| 能力 | 说明 |
|------|------|
| **可视化编排** | 拖拽节点，连接线条，像画流程图一样设计 AI 应用 |
| **多模型支持** | 接入 OpenAI、Claude、Gemini、Mistral 等几十种大模型 |
| **知识库管理** | 上传你的文档，让 AI 基于你的私有数据回答问题 |
| **应用发布** | 一键生成网页应用、API 接口，方便集成到现有系统 |
| **运营监控** | 查看使用日志、分析对话质量、持续优化效果 |
| **开源免费** | 核心功能完全开源，可以私有化部署，数据完全自主可控 |

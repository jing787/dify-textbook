# 第 12 章：新场景——多平台内容工厂

> 小林解决了客服和周报两个痛点，现在轮到第三个：**营销内容**。
>
> 每当有新品上市，市场部就会来找小林："帮忙写一下产品介绍，要发 Blog、Instagram、Twitter/X，还有 LinkedIn……"
>
> 同一个产品，要写四五个版本。每个平台风格不同：
> - **Blog**：正式、详细
> - **Instagram**：视觉化、种草
> - **Twitter/X**：简短、有梗
> - **LinkedIn**：专业、商务
>
> "能不能输入一份产品资料，自动生成各平台的内容？"

## 设计内容工厂 Workflow

这个场景的特点：
- 一个输入，多个输出（不同平台）
- 每个输出都需要特定的风格调整

**实现方式：迭代生成**

定义平台列表，用迭代节点循环生成。将来添加新平台只需要修改列表。

## 第一步：定义输入和平台列表

创建新 Workflow："TechStore 内容工厂"

**开始节点配置：**

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `product_info` | 长文本 | 产品资料 |
| `platforms` | 字符串数组 | 目标平台列表 |
| `tone_notes` | 文本 | 额外的语气要求（可选）|

`platforms` 默认值：`["Blog", "Instagram", "Twitter/X", "LinkedIn"]`

## 第二步：构建迭代流程

**添加迭代节点：**

1. 配置迭代输入：选择「开始 / platforms」
2. 进入迭代子画布

**在迭代内部添加 LLM 节点：**

```
你是一位资深的内容营销专家。请根据产品资料，为指定平台撰写营销内容。

产品资料：
{{ product_info }}

目标平台：{{ item }}

各平台风格指南：

【Blog】
- 语气：专业、可信赖
- 长度：800-1200 words
- 结构：标题 + 导语 + 产品介绍 + 使用场景 + 购买引导
- 特点：SEO-friendly，结构清晰，可以用 emoji 点缀

【Instagram】
- 语气：lifestyle 种草、真实分享
- 长度：300-500 words
- 结构：吸睛标题 + 使用体验 + 亮点罗列 + 购买建议
- 特点：大量 emoji，口语化，带 #hashtag

【Twitter/X】
- 语气：轻松有趣、有梗
- 长度：100-200 words
- 结构：一句话 hook + 核心卖点 + 互动引导
- 特点：简短精悍，带 #话题标签

【LinkedIn】
- 语气：专业商务
- 长度：200-400 words
- 结构：行业洞察 + 产品价值 + 商务邀请
- 特点：突出专业性和商业价值

请根据平台特点，输出合适的营销内容。
```

## 第三步：整理输出

迭代完成后，用模板转换节点整理格式：

```jinja2
===================================
     TechStore 内容生产报告
===================================

产品：{{ product_name }}
生成时间：{{ timestamp }}

{% for i in range(platforms|length) %}
-----------------------------------
【{{ platforms[i] }}】
-----------------------------------
{{ contents[i] }}

{% endfor %}
===================================

内容已准备就绪，可直接复制使用。
如需调整，请修改后重新生成。

===================================
```

## 进阶：加入多语言

如果还需要生成多语言版本？

简单的方式：把"语言"也放进平台列表：

```
platforms: ["Blog-English", "Instagram-English", "LinkedIn-English", "Blog-Japanese", "Blog-Korean"]
```

然后在 Prompt 中识别语言要求：

```
根据平台名中标注的语言撰写。
例如 "Blog-Japanese" 用日语，"Blog-Korean" 用韩语，默认用英语。
```

这样一个 Workflow 就能处理**多平台、多语言**的内容生成。

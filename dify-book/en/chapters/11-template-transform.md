# Chapter 11: Format Control — The Template Transform Node

> The weekly report Workflow worked great, but Lynn noticed an issue:
>
> The report format wasn't always consistent. Sometimes it used Markdown, sometimes it mixed in other formats. Sometimes data appeared in tables, other times in lists.
>
> "I want the output format to always be the same — no more variation every time."
>
> That's where the "Template Transform" node comes in.

## What Is Template Transform

The Template Transform node lets you format data using a **fixed template**.

It uses **Jinja2** template syntax — a simple yet powerful templating language.

**Basic concepts:**
- **Variables**: Represented by `{{ variable_name }}`, replaced with actual values
- **Loops**: Use `{% for item in list %}` to iterate over lists
- **Conditionals**: Use `{% if condition %}` for conditional logic

**Example:**

Template:
```
Dear {{ customer_name }},
Your order {{ order_id }} has been shipped.
```

Input:
```
customer_name = "John Smith"
order_id = "2025001"
```

Output:
```
Dear John Smith,
Your order 2025001 has been shipped.
```

With Template Transform, you can ensure the output format is always consistent.

## Applying Templates to the Weekly Report

Modify the weekly report Workflow:

1. After the "Report Generation" LLM node, add a "Template Transform" node
2. Configure the Template Transform:

**Input variables:**
- `report_content`: Output from the Report Generation node
- `week_range`: Date range from the Start node

**Template content:**

```
=====================================
      TECHSTORE WEEKLY OPERATIONS REPORT
=====================================

Report Period: {{ week_range }}
Generated At: {{ current_time }}

-------------------------------------

{{ report_content }}

-------------------------------------

[Note]
This report was automatically generated by AI using data from various business systems.
For questions, please contact the Operations department.

=====================================
```

3. Connect the Template Transform output to the End node

Now, regardless of what the LLM generates, the final output will always be wrapped in this fixed format.

## Working with List Data

Template Transform is particularly useful when working with list data.

For example, in the customer service scenario, an Iteration node outputs a list of answers. We can use Template Transform to format them:

```jinja2
Dear Customer,

Thank you for your inquiry. Here are the answers to your questions:

{% for i in range(answers|length) %}
[Question {{ i + 1 }}]
{{ questions[i] }}

[Answer]
{{ answers[i] }}

{% endfor %}

If you have any other questions, feel free to reach out anytime.

TechStore Customer Service Team
```

**Input:**
```
questions = ["Is it available in white?", "Any current promotions?", "Can I return it?"]
answers = ["Yes, ...", "We currently have...", "Yes, ..."]
```

**Output:**
```
Dear Customer,

Thank you for your inquiry. Here are the answers to your questions:

[Question 1]
Is it available in white?

[Answer]
Yes, the TechPhone Pro 16 comes in Glacier White...

[Question 2]
Any current promotions?

[Answer]
We currently have a trade-in promotion...

[Question 3]
Can I return it?

[Answer]
Yes, unopened items can be returned within 7 days...

If you have any other questions, feel free to reach out anytime.

TechStore Customer Service Team
```

## Jinja2 Syntax Quick Reference

**Variable output:**
```jinja2
{{ variable }}
```

**Conditionals:**
```jinja2
{% if score > 90 %}
Excellent
{% elif score > 60 %}
Pass
{% else %}
Fail
{% endif %}
```

**Loops:**
```jinja2
{% for item in list %}
- {{ item }}
{% endfor %}
```

**Loops with index:**
```jinja2
{% for item in list %}
{{ loop.index }}. {{ item }}
{% endfor %}
```

**Filters (processing variables):**
```jinja2
{{ text | upper }}        → Convert to uppercase
{{ text | lower }}        → Convert to lowercase
{{ text | length }}       → Get length
{{ list | join(", ") }}   → Join list with commas
```

**Default values:**
```jinja2
{{ name | default("Anonymous User") }}
```
